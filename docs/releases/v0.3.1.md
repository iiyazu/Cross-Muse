# xmuse v0.3.1

`v0.3.1` is a production-evidence and quality release for the local, Room-first xmuse
Workroom. It does not expand the authority model or add a remote collaboration protocol.

## Source-backed memory evidence

The full-local MemoryOS dogfood used two four-Agent Rooms and proved an approved project
source could be recalled outside the current correlation, causal envelope, and recent Room
burst, then re-proved against its originating Room activity in `chat.db`.

- 11 correlations settled;
- one approved project candidate was delivered and recalled across Rooms;
- six scope bindings were attached and both message/document backlogs drained to zero;
- one non-empty Room B receipt was bound to the same context digest;
- two derived page/recall items retained re-provable sources;
- MemoryOS recovered as a single child after SIGKILL;
- zero unapproved cross-Room sources, browser-console errors, or sensitive leaks.

Safe result SHA-256:
`e29ddeadeabf7e90bf7edb675ad7409fa5471432c639cc3dd69dfa680575d746`.

## Exact-patch repository matrix

The server-owned Harness profiles were exercised against frozen repository trees. Exact
patches were promoted in memU with `python-uv/v1`, Clowder with `docs/v1`, and MemoryOS as the
control. Letta failed closed when the fixed Python toolchain capability was unavailable, and
mem0 failed closed when its profile marker was missing. No repository supplied gate argv or
triggered an online dependency install.

Safe matrix SHA-256:
`b5d519d3f053865ef2d92df06962169565cbc9fe65737705ebcc029774f3fe9d`.

## Runtime boundaries

The former wide memory runtime dependency is split into three capabilities:

- Room Host receives recall and recall-receipt operations;
- Room Codex transport receives only context-receipt binding;
- Runner composition receives only the delivery pump.

The same enabled or disabled runtime can implement all three ports. `chat.db`, recall source
proof, caller-owned transactions, MemoryOS product behavior, and public projections are
unchanged.

## 120-minute provider endurance

The release candidate ran eight Rooms with two persistent Codex Agents for five waves under
full-local MemoryOS. The run injected an app-server SIGKILL, a Runner SIGKILL under four
active deliveries, a MemoryOS SIGKILL, and a private stream-cache deletion.

- 40/40 Human correlations settled;
- 172 attempts, below the fixed cap of 192;
- maximum four concurrent provider deliveries;
- fault recovery: Codex 0.6 s, Runner 7.0 s, MemoryOS 8.3 s, stream cache 11.1 s;
- zero duplicate outcome, cross-Room identity/causality, provider orphan, live lease,
  cleanup, recovery, exhausted, or incomplete-attempt residue;
- 115 memory outbox deliveries, 59 recall receipts, and 114 source references;
- FD growth 0 and RSS growth 0 after warm-up;
- SQLite integrity `ok`, browser console errors 0, and repository bytes unchanged;
- second-half settle p95 was 206.95 s versus 317.07 s in the first half.

Safe result SHA-256:
`f261ccb6afefc15a824521745dae1e13036b1e1759bb815505772939fbdb98b2`.
Raw prompts, provider output, databases, logs, and screenshots are not release artifacts.

## Product improvements since v0.3.0

- narrower frontend subscriptions and smoother long timelines, streaming previews, Dock
  activity, Markdown, focus, and reduced-motion behavior;
- staged Room outcome/Host/Codex boundaries without splitting durable SQLite transactions;
- measured Room read-model behavior and production dogfood coverage;
- guarded source-backed recall, external-repository Harness qualification, and bounded
  endurance recovery evidence.

## Supported boundary

- Linux x86_64 / WSL with CPython 3.11;
- loopback-only, single-user deployment;
- authenticated local Codex CLI;
- MemoryOS remains opt-in and its index remains derived/rebuildable;
- Room Codex stays read-only, network-disabled, and limited to the single durable outcome
  MCP tool;
- Google A2A remains retired and is not part of the Room main path;
- no remote authentication, native Windows/macOS bundle, Dashboard, central speaker queue,
  or general process manager.
